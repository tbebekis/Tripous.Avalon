AUTO ITEMPAGE LAYOUT RULES
==========================

Terms
-----
TopTable  = ModuleDef.Table
TopFields = bindable fields of TopTable
Detail    = child table of TopTable
SubDetail = child table of a Detail or another SubDetail

General Rules
-------------
1. TopFields always appear first.

2. TopFields are grouped by FieldDef.Group.

3. Each TopFields group appears inside its own Expander.

4. Empty FieldDef.Group is treated as "General".

5. Inside each TopFields group, fields are split into 1..3 columns using FormDef.ColumnCount.

6. Boolean fields may occupy the full row.

7. Only bindable fields are displayed.

8. Memo TopFields are displayed as multi-line editors.
   Height is based on Ui.MemoRowCount, clamped between 3 and 5 rows.

9. Image TopFields are displayed as full-row image preview controls.
   The preview uses fixed height, keeps aspect ratio, and may provide Load/Clear actions.

10. Detail grids are NEVER placed inside Expanders.

11. Memo and Image fields are NOT displayed as automatic DataGrid columns
    in Detail/SubDetail grids, even if visible/bindable.

Top Table Layout Rules
----------------------
12. If TopTable has no Details:
    The ItemPage contains only TopFields groups.

13. If TopTable has Details:
    Create a root TabControl.

14. First TabItem of the root TabControl = TopTable page.

15. The TopTable page contains:
    - TopFields groups
    - optional Level 1 Detail TabControl under the TopFields

16. All first-level Details of TopTable appear inside the Level 1 Detail TabControl.

17. Each first-level Detail appears in its own TabItem.

18. First-level Detail tabs contain:
    - optional toolbar panel
    - DataGrid or one-to-one controls

SubDetail Rules
---------------
19. Level 2+ Details/SubDetails are added directly to the root TabControl
    as sibling TabItems of the TopTable page.

20. Each Detail/SubDetail appears in its own TabItem.

21. One-to-many Detail/SubDetail is displayed using a DataGrid.

22. One-to-one Detail/SubDetail (TableDef.IsOneToOne = true)
    is displayed using grouped controls instead of a DataGrid.

23. Detail/SubDetail tabs may contain:
    - optional toolbar panel
    - DataGrid or controls

Toolbar Rules
-------------
24. Each Detail/SubDetail may optionally contain a toolbar area
    above the DataGrid.

25. Toolbar structure:

    Border Classes="ToolbarContainer"
        StackPanel Classes="ToolBar"

26. Toolbar panels are initially hidden and may later host buttons,
    filters, locator controls, summaries, exports, etc.

Overridables
------------
27. Expanders are created through an overridable method:
    CreateExpander(Control Parent, string Caption)

Core Principle
--------------
TopFields = controls.
One-to-many Detail/SubDetail = DataGrid.
One-to-one Detail/SubDetail = controls.
Detail grids never inside Expanders.
